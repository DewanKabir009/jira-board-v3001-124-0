          Signals the AI pass must consider

          • Main tickets with hidden or stale subtasks
          • Component clusters with repeated failures

          Safety contract

          Review gates before AI touches work systems

          1. Use cited board, ticket, automation, and knowledge-base sources in every generated summary.
          2. Keep prompts, model settings, and source selection behind the QA Admin role.
          3. Treat every AI output as a draft until a named user approves it.
          4. Do not send Jira comments, Slack messages, or automation runs directly from AI output.
          5. Log generated summary metadata so stale or uncited summaries can be detected.
